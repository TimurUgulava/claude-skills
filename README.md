# Claude Skills

Коллекция кастомных скиллов для [Claude Code](https://docs.anthropic.com/en/docs/claude-code).

## Skill Architect

**Архитектор скиллов** — проектирует и собирает промышленные скиллы для Claude Code по 4-фазному протоколу.

### Что делает

Берёт техническое задание (или просто описание задачи словами) и превращает его в готовый скилл: директория с `SKILL.md`, набор `references/` и упакованный `.skill` файл.

### 4 фазы работы

| Фаза | Что происходит |
|------|---------------|
| **Discovery** | Выясняет задачу, триггеры, формат вывода. Задаёт уточняющие вопросы |
| **Blueprint** | Выбирает архитектурный паттерн, рисует Mermaid-схему, предлагает структуру |
| **Compilation** | Собирает SKILL.md + references, упаковывает в `.skill` |
| **Debug** | Если что-то работает не так — предлагает протестировать и докручивает |

### Принципы

- **Атомарная детализация** — каждое требование декомпозируется на проверяемые инструкции
- **SKILL.md ≤ 100 строк** — ядро в body, детали в references. Экономит токены при каждом триггере
- **Архитектор, не исполнитель** — создаёт скилл для задачи, а не выполняет задачу сам
- **Генерализация** — скилл должен работать тысячи раз, не только на примерах из ТЗ

### Установка

Скопируйте директорию `skill-architect/` в ваш проект:

```bash
cp -r skill-architect/ /path/to/your/project/.claude/skills/skill-architect/
```

Или установите из `.skill` файла (если поддерживается вашей версией Claude Code).

### Как использовать

Скажите Claude Code что-то вроде:

- «Создай скилл для анализа PR»
- «Мне нужен скилл, который генерирует отчёты»
- «Спроектируй скилл для code review»

Skill Architect сам проведёт вас через все фазы.

### Структура

```
skill-architect/
├── SKILL.md                              # Ядро скилла
├── references/
│   ├── protocol.md                       # Фазы Discovery и Blueprint

```

## Calibrator

**Калибровщик промптов и ассистентов** — инженерная петля обратной связи для любых инструкций. Сравнивает AI-версию результата с авторской финальной редакцией, находит системные паттерны расхождений и формулирует конкретные правки в инструкцию.

### Что делает

Любой ассистент выдаёт 70–80% результата, остальное дорабатывается руками. И каждый раз — одни и те же правки. Калибровщик замыкает петлю: берёт две версии (AI и финальную), достаёт из дельты повторяющиеся паттерны, превращает их в точные дополнения к промпту. Через 5–10 итераций ассистент выдаёт ближе к финалу с первой попытки.

Работает с любыми ассистентами — текстовыми, кодовыми, аналитическими. Где есть «AI-версия» и «как должно быть» — там строится калибровка.

### Как устроена работа

| Шаг | Что происходит |
|-----|---------------|
| **1. Подготовка + анализ** | Читает текущую инструкцию и CHANGELOG, сверяет две версии блок за блоком |
| **2. Классификация** | Каждое расхождение — единичная правка / системный паттерн / рецидив |
| **3. Формулировка** | Для каждого паттерна — точный текст правки с примером «до/после» |
| **4. Применение** | Вносит правки в файл + обновляет CHANGELOG **или** выдаёт готовый блок для копирования |

### Два режима работы

Скилл сам определяет окружение и подстраивается:

- **Режим A (файловый)** — в Claude Code и VS Code, где есть доступ к файлам. Читает инструкцию, вносит правки прямо в файлы, ведёт `CHANGELOG.md`.
- **Режим B (диалоговый)** — в веб-чате Claude, ChatGPT, мобильном приложении. Автор вставляет две версии текстом, скилл возвращает таблицу паттернов и готовые формулировки правок для ручной вставки.

Один и тот же `SKILL.md` можно:
- подключить как скилл в Claude Code,
- загрузить инструкцией в Claude Project на claude.ai,
- вставить body как system prompt в любой LLM.

### Принципы

- **Сдержанность** — не каждая авторская правка превращается в правило. Паттерн = 2+ повторений или рецидив из истории
- **Конкретность** — правка = точный текст для вставки в инструкцию. Никаких «пиши естественнее»
- **Охота за рецидивами** — если правило уже было, но не сработало, диагностируй причину, а не добавляй третье поверх двух неработающих
- **Доменная нейтральность** — не привязан к конкретной области, работает с любым типом инструкций

### Установка

```bash
cp -r calibrator/ ~/.claude/skills/calibrator/
```

Или скопируйте в конкретный проект: `/path/to/project/.claude/skills/calibrator/`.

### Как использовать

Скажите Claude что-то вроде:

- «Откалибруй мой промпт — вот AI-версия, вот моя финалка»
- «Сравни две версии и предложи правки в инструкцию»
- «Почему ассистент всё время пишет одно и то же, как поправить скилл»

Скилл сам выяснит, какую инструкцию калибруем, и проведёт через все 4 шага с точками остановки.

### Структура

```
calibrator/
└── SKILL.md         # Самодостаточный протокол — весь скилл в одном файле
```
