# Calibrator

**Калибровщик промптов и ассистентов** — инженерная петля обратной связи для любых инструкций. Сравнивает AI-версию результата с авторской финальной редактурой, находит системные паттерны расхождений и формулирует конкретные правки в инструкцию.

## Зачем

Любой ассистент выдаёт 70–80% результата, остальное дорабатываешь руками. И каждый раз — одни и те же правки. Одни и те же клише, те же пропуски, те же структурные косяки.

Калибровщик замыкает петлю: берёт две версии (AI и финальную), достаёт из дельты повторяющиеся паттерны, превращает их в точные дополнения к промпту. Через 5–10 итераций ассистент выдаёт ближе к финалу с первой попытки. Остаются единичные правки — на них всё равно нужен человек.

Работает с любыми ассистентами: текстовыми, кодовыми, аналитическими, с промптами, планами. Где есть «AI-версия» и «как должно быть» — там строится калибровка.

## Как устроена работа

Четыре шага с точками остановки. Автор утверждает переход между каждым.

| Шаг | Что происходит |
|-----|---------------|
| **1. Подготовка + анализ** | Читает текущую инструкцию и CHANGELOG, сверяет две версии блок за блоком по 8 универсальным категориям (объём, структура, форма, детализация, точность, контекст, конвенции, вход/выход) |
| **2. Классификация** | Каждое расхождение — единичная правка / системный паттерн / рецидив. Единичное — фиксируется, но в правила не идёт. Системное (2+ повторений) — кандидат на правку. Рецидив — усиленный сигнал, значит, предыдущая правка не сработала |
| **3. Формулировка** | Для каждого паттерна — точный текст правки: что вставить, куда вставить, пример «до/после» из разобранной работы |
| **4. Применение** | Вносит правки в файл инструкции + обновляет CHANGELOG **или** выдаёт готовый блок для копирования — в зависимости от режима (см. ниже) |

## Два режима работы

Скилл сам определяет окружение и подстраивается.

### Режим A — файловый (Claude Code, VS Code)

Доступны Write/Edit. Работает по полному протоколу:
- читает текущую инструкцию из файла
- вносит одобренные правки прямо в файл
- ведёт `CHANGELOG.md` рядом с инструкцией (создаёт, если его нет)

### Режим B — диалоговый (веб-чат claude.ai, ChatGPT, мобильное приложение)

Файловых инструментов нет. Автор вставляет две версии текстом, скилл возвращает:
- таблицу выявленных паттернов (единичные / системные / рецидивы — раздельно)
- готовые формулировки правок с указанием, куда каждую вставлять
- готовый markdown-блок для CHANGELOG
- если попросить — собранный целиком обновлённый текст инструкции

Один и тот же `SKILL.md` можно:
- подключить как скилл в Claude Code;
- загрузить инструкцией в Claude Project на claude.ai;
- вставить body как system prompt в любой LLM.

## Принципы

- **Сдержанность.** Не каждая авторская правка превращается в правило. Паттерн = 2+ повторений или рецидив из истории. Гиперреакция — главная ошибка калибровщика
- **Конкретность.** Правка = точный текст для вставки в инструкцию. Никаких «делай лучше», «пиши живее», «следуй best practices» — модель на абстракции не реагирует. Только дословный текст с примером «до/после»
- **Охота за рецидивами.** Если правило уже было, но не сработало — диагностируй причину (слишком абстрактное? не там стоит? без примера? противоречит другому?), а не добавляй третье поверх двух неработающих
- **Доменная нейтральность.** Не привязан к конкретной области. Категории анализа работают одинаково для текстов, кода, аналитических отчётов, промптов, планов

## Установка

### Для Claude Code (user-level, во всех проектах)

```bash
cp -r calibrator ~/.claude/skills/
```

### Для Claude Code (внутри проекта)

```bash
cp -r calibrator /path/to/project/.claude/skills/
```

### Для Claude Project на claude.ai

Скопируйте содержимое `SKILL.md` в инструкции проекта (Project Instructions).

### Для обычного чата (ChatGPT, веб-Claude, мобильное приложение)

Скопируйте тело `SKILL.md` (всё после frontmatter) и вставьте как system prompt или в начало диалога: «Действуй по инструкции ниже: …».

## Как использовать

Скажите ассистенту что-то вроде:

- «Откалибруй мой промпт — вот AI-версия, вот моя финалка»
- «Сравни две версии и предложи правки в инструкцию»
- «Почему ассистент всё время пишет одно и то же, как поправить скилл»
- «AI выдал X, я поправил на Y — разберись, что не так с промптом»

Скилл сам выяснит, какую инструкцию калибруем, и проведёт через все 4 шага с точками остановки.

### Пример входа

1. **AI-версия** — то, что выдал ассистент
2. **Авторская финальная версия** — результат после ручной редактуры
3. **Инструкция, которую калибруем** — текущий промпт / SKILL.md / system prompt
4. **(опционально) CHANGELOG** — история прошлых итераций, чтобы ловить рецидивы

## Что НЕ делает

- Не пишет текст/код с нуля — это задача для writer-скиллов
- Не делает разовую правку одного результата — это корректура, не калибровка
- Не проверяет факты — это отдельная задача (fact-checker)
- Не реагирует на единичное стилистическое предпочтение — только на системные паттерны
