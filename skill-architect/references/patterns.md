# Архитектурные паттерны скиллов

## Содержание
1. Sequential Workflow
2. Iterative Refinement
3. Context-Aware Selection
4. Domain Intelligence
5. Multi-MCP Coordination
6. Антипаттерны

---

## 1. Sequential Workflow

**Когда:** Чёткий пошаговый процесс, каждый шаг зависит от предыдущего.

```markdown
## Алгоритм
1. Извлеки входные данные из запроса
2. Валидируй по схеме (references/schema.md)
3. Если недостаточно — запроси через AskUserQuestion
4. Трансформируй: {{описание}}
5. Применить шаблон вывода
6. Проверь по чек-листу
```

**Степень свободы:** Низкая. Каждый шаг детерминирован.

---

## 2. Iterative Refinement

**Когда:** Результат улучшается за несколько циклов.

```markdown
## Алгоритм
1. Создай первичную версию
2. Оцени по критериям: {{список}}
3. Если оценка < порога → определи слабые места → исправь → шаг 2
4. Финализируй
```

**Степень свободы:** Средняя. Критерии жёсткие, способ — гибкий.

---

## 3. Context-Aware Selection

**Когда:** Одна цель, разные подходы по контексту.

```markdown
| Сигнал | Контекст | Подход |
|--------|----------|--------|
| .xlsx | Табличные данные | openpyxl |
| .csv | Простые данные | pandas |
| URL | Внешний источник | WebFetch → парсинг |
```

**Степень свободы:** Средняя.

---

## 4. Domain Intelligence

**Когда:** Нужны специализированные знания.

```markdown
## Доменная база знаний
### Ключевые концепции
- Термин: определение

### Правила домена
1. Правило с обоснованием *почему*

Расширенная документация: references/domain-knowledge.md
```

**Степень свободы:** Низкая для правил, средняя для генерации.

---

## 5. Multi-MCP Coordination

**Когда:** Workflow через несколько внешних сервисов.

```markdown
## MCP-серверы
- server_1: назначение
- server_2: назначение

## Обработка ошибок
| Сервер | Ошибка | Fallback |
|--------|--------|----------|
| server_1 | Timeout | Повтор ×3 |
```

**Степень свободы:** Низкая.

---

## 6. Антипаттерны

| Антипаттерн | Проблема | Решение |
|-------------|----------|---------|
| Workflow в description | Агент следует description, игнорирует body | Description — только триггеры |
| Монолитный SKILL.md (>500 строк) | Медленная загрузка | Детали → references/ |
| Нет negative triggers | Ложные срабатывания | Всегда "Do NOT use for..." |
| Абстрактные инструкции | Агент интерпретирует по-разному | Атомарные шаги |
| MUST/NEVER без обоснования | Нарушается при неоднозначности | Объясняй *почему* |
