# Фаза 3: Сборка скилла (Compilation)

Создай файлы скилла через Write/Edit.

## 1. Frontmatter

```yaml
---
name: kebab-case-name          # Строчные, цифры, дефисы. Max 64 символа
description: >
  [Назначение в одном предложении].
  Use when [триггерные фразы в кавычках].
  Do NOT use for [negative triggers].
---
```

**Правила description (от этого зависит триггеринг):**
- Max 1024 символов, без угловых скобок `< >`
- Формула: назначение → "Use when..." → "Do NOT use for..."
- Конкретные фразы пользователя в кавычках
- НИКОГДА workflow/шаги в description — агент будет следовать description вместо body
- Делай description «настойчивым» — скиллы склонны к undertriggering

## 2. Body структура

Шаблоны → `references/skill-template.md`. Обязательные секции:

1. **Обзор** — что делает, 1–2 предложения
2. **Алгоритм** — пронумерованные атомарные шаги верхнего уровня, императивный залог
3. **Указатели на references** — где детали, шаблоны, примеры
4. **Ограничения** — что НЕ делает, кратко

Всё остальное — в references/:
- Шаблоны вывода → `references/output-templates.md`
- Примеры (Ввод → Результат) → `references/examples.md` или `examples/`
- Доменные знания → `references/domain.md`
- Обработка ошибок → `references/errors.md`

## 3. Правила написания

- **Один термин на концепт.** «Шаблон» везде — не шаблон/скелет/каркас
- **Императивный залог.** «Извлеки», «Проверь» — не «Ты должен извлечь»
- **Explain the why.** Вместо «ВСЕГДА JSON» → «JSON, потому что downstream pipeline парсит только его»
- **SKILL.md ≤ 100 строк.** Body = ядро + указатели. Причина: загружается целиком при триггере, раздутый файл съедает токены
- **Прогрессивное раскрытие.** SKILL.md → «подробности в references/X.md». Один уровень вложенности
- **Бандли повторяющееся.** Детерминированная операция → скрипт в scripts/

## 4. References

- Markdown, оглавление для файлов > 300 строк
- Укажи назначение каждого файла в SKILL.md
- Содержимое: детальные инструкции, шаблоны, примеры, доменные знания

## 5. Scripts

- Python или Bash с `#!/usr/bin/env python3` / `#!/usr/bin/env bash`
- Descriptive stdout/stderr при ошибках
- Без захардкоженных путей/токенов

## Бюджет контекста

| Уровень | Загрузка | Бюджет |
|---------|----------|--------|
| Frontmatter | Всегда | ~100 слов |
| SKILL.md body | При триггере | ≤ 100 строк |
| references/ | По требованию | Без лимита |
| scripts/ | Исполняются | Без лимита |
| assets/ | Никогда | Без лимита |
